## 🤔 为什么 RSS 在算法时代反而更重要 (Why RSS)

> 💡 **核心理由**：① 没有算法干预 · ② 没有广告打扰 · ③ 没有平台绑架 · ④ 跨平台跨终端通用

信息爆炸时代，「找到值得读的内容」反而比「能不能读到」更难——算法推荐让你陷入信息茧房，社交媒体让你被无限滚动绑架，AI 生成内容让信噪比急剧恶化。

**RSS 是翔宇这些年一直在用的核心工具**。你订阅什么、读什么完全由你决定：

- 🎯 **没有算法干预** — 信息按时间线展示，你按需要消费
- 🚫 **没有广告打扰** — 干净的内容流，专注阅读
- 🔓 **没有平台绑架** — 平台改规则、改算法都带不走你的源
- 🌐 **跨平台跨终端** — 同一份 OPML 在所有阅读器通用

> 💎 **RSS 是信息获取与筛选的主干通道。** 翔宇用 RSS 构建私人信息网络已超过两年，配合 AI 编程工具（Claude Code / Codex）和自动化平台（Make / n8n）做摘要、监控、翻译、选题。

这个仓库做的就是把全网值得订阅的优质源集中索引、按读者场景分类，让你不用四处搜索、不用从零起步——**直接挑一个分类导入阅读器，30 分钟搭好私人信息网。**

## 📥 怎么导入到 RSS 阅读器 (Import OPML)

下载 OPML 文件后，按你用的阅读器导入：

| 阅读器 | 导入路径 |
|---|---|
| **Inoreader** | Settings → Import / Export → Import OPML |
| **Feedly** | Settings → Import OPML |
| **NetNewsWire** | File → Import Subscriptions |
| **Miniflux**（自建）| Settings → Import → OPML file |
| **FreshRSS**（自建）| 配置 → Import / Export → 导入 OPML |
| **Readwise Reader** | Settings → Import → OPML |

> 🎯 **第一次导入建议**：
>
> 1. 不要一开始就导入主 `feeds.opml`（2000 源完整合集可能让阅读器卡顿）
> 2. 按兴趣挑 1-2 个分类 OPML 导入
> 3. 试读一周，删掉不感兴趣的，再导入下一个分类

## 🤖 配合 Claude Code / Codex 使用 (AI Workflows)

> 🚀 **RSS 是 AI 编程工具最好的信息输入。** Claude Code / Codex 本身就能跑 Python——把本仓库的 OPML 直接喂给它，几句话就能搭起信息处理流水线，**零部署、零账号、零接口密钥**。

### 🌅 场景 1：每日 AI 摘要日报（零部署版）

直接让 Claude Code 或 Codex 从 GitHub 抓 OPML，feedparser 解析，AI 摘要：

```text
你（对 Claude Code 或 Codex）：

  下载 https://raw.githubusercontent.com/xiangyugongzuoliu/awesome-rss-feeds/main/feeds/cn-essays.opml，
  用 feedparser 解析里面所有源，抓取最近 24 小时的文章正文，
  用中文写 500 字摘要日报，按主题分组（思考 / 文化 / 技术 / 时事），
  保存到本地笔记目录。
```

🌅 **每天起床打开就有一份属于你的中文技术圈日报。不需要任何中间服务。**

### 📊 场景 2：竞品 / 行业监控（定时任务 + 推送）

让 Codex 写一个独立 Python 脚本，注册定时任务跑：

```text
你（对 Codex）：

  写一个 Python 脚本：抓 feeds/en-startups.opml 里所有源，
  保存当前文章 ID 去重。下次运行时找出新增文章，
  用关键词过滤"产品发布 / 定价变化 / 融资 / 收购"，
  命中的摘要 + 原文链接推送到本地笔记。

  最后注册 cron：每天早 8 点跑一次。
```

⏰ **每月省下两小时手动盯竞品的时间。**

### 📝 场景 3：写作选题库

```text
你（对 Claude Code 或 Codex）：

  从 feeds/cn-essays.opml + feeds/cn-podcasts.opml 最近一周文章里，
  用 AI 提取热点话题，按出现频次排序，写入云笔记。
  每个话题含：相关文章数、代表标题 3 条、首次出现日期。
```

✍️ **写公众号 / 博客 / 视频时打开笔记就有热点选题候选。**

### 🌐 场景 4：英文 RSS 中文化

英文圈优质内容 + AI 翻译 = 私人英文信息中文版：

```text
你（对 Claude Code 或 Codex）：

  抓 feeds/en-ai-research.opml + feeds/en-web-frontend.opml 这周的英文文章，
  AI 翻译成中文摘要（保留专业术语原文），写一份周报，
  归档到云笔记。
```

🌍 **英文圈最新进展，中文摘要直达。**

### 🧠 场景 5：本地 RAG 知识库

把所有 RSS 文章内容做向量化，做你自己的「领域内问答」：

```text
你（对 Claude Code 或 Codex）：

  从 feeds/en-ai-research.opml + feeds/en-ai-tools.opml 持续抓取全文，
  喂入本地 ChromaDB / pgvector，做向量索引。
  我提问"最近 RAG 有什么新进展"时检索 + AI 回答。
```

🧠 **把 RSS 沉淀成你的私人知识库——比公网搜索精准，因为是你筛过的源。**

### 🛠️ 推荐工具链（按需要的复杂度递进）

| 阶梯 | 工具 | 适合谁 |
|---|---|---|
| 🟢 **入门**（零部署） | **Claude Code / Codex + Python `feedparser`** | 大多数人 — 本仓库 OPML 直接抓，一句话搞定 |
| 🟡 **自动化**（定时触发） | + **cron / launchd** 或 **n8n / Make** | 想让 RSS-AI 流水线每天自动跑 |
| 🟠 **进阶**（状态管理 + 多端） | + **Miniflux / FreshRSS** 自建 + REST API | 想要"已读 / 未读 / 增量更新"，多端同步 |
| 🔴 **专业**（向量索引 + RAG） | + **ChromaDB / pgvector / Qdrant** | 把 RSS 沉淀成可问答的知识库 |
| ✨ **补充**（无 RSS 网站转 RSS） | **RSSHub / RSS-Bridge** | 想订阅微博 / 推特 / 抖音等没原生 RSS 的源 |
| 📦 **输出归档** | **Notion / Obsidian / Logseq** | 长期沉淀 + 跨设备访问 |

> 💡 **不用一上来就部署 Miniflux**——绝大多数场景，**Claude Code / Codex + feedparser + 本仓库 OPML** 就够了。等真的有「我要管理几百个源 + 多端同步状态」的需求时再升级。
